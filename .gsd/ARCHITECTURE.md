## Overview

PEML (Programming Exercise Markup Language) is a Ruby-based project designed to provide a simple, easy format for instructors to describe programming assignments and activities. It includes a parser, a testing DSL (PEMLtest), and tools to convert these descriptions into executable tests.

## Components

### Core PEML Module
- **Purpose:** The main entry point for the library. Provides methods like `Peml.parse` and `Peml.validate`.
- **Location:** `lib/peml.rb`
- **Dependencies:** `dottie`, `json_schemer`

### PEML Loader
- **Purpose:** A custom scanner/parser that processes the high-level PEML structure (key-value pairs, scopes, arrays).
- **Location:** `lib/peml/loader.rb`

### PEMLtest Parser
- **Purpose:** A `parslet`-based parser for the PEMLtest domain-specific language used to define test cases.
- **Location:** `lib/peml/parser.rb`

### Datadriven Test Renderer
- **Purpose:** Generates language-specific test code (Ruby, Python, Java, C++) from tabular data in PEML files using Liquid templates.
- **Location:** `lib/peml/datadriven_test_renderer.rb`

### PIF Subsystem
- **Purpose:** Parsing and converting PIF (Parsons Problems Input File) descriptions.
- **Location:** `lib/pif/`

## Data Flow

1. Text content is passed to `Peml.parse` or `PifParser.parse`
2. The `Loader` converts the text into a Ruby Hash
3. The structure is validated using `json_schemer` against JSON Schemas (`PEML.json` or `PIF.json`)
4. Data passes through various internal transformation methods (`TRANSFORMS` in `lib/peml.rb`)
5. Data is returned or passed to Liquid templates for generation
